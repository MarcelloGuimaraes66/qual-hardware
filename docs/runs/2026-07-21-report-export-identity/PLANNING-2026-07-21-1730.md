# Planejamento - identidade inequívoca dos relatórios

## Problema

A interface mostra dois botões chamados `PDF`. O segundo baixa o anexo técnico neutro de 31 páginas, enquanto o primeiro baixa o relatório comparativo restaurado. O primeiro grupo também está rotulado incorretamente como `Relatório comercial + neutro` e salva o PDF com um nome diferente do nome retornado pela API.

## Meta

Tornar impossível confundir o relatório completo de recomendações com o anexo técnico neutro.

## Controles T2

- Invariantes: geradores, banco, cálculo, API e conteúdo dos documentos permanecem inalterados.
- Blast radius: controles de exportação, nomes locais dos downloads, mensagens e teste dirigido.
- Orçamento: `fix + minimal refactor`.
- Plataformas: interface e download usam o mesmo código em macOS, Windows e Ubuntu.
- Rollback: reverter o commit local; nenhum dado persistido é tocado.

## Aceitação

- Um único botão primário visível chamado `Baixar relatório PDF`.
- Esse botão salva `qual-hardware-recomendacoes.pdf`.
- O anexo neutro fica em uma área recolhida, explicitamente descrita como documento separado.
- Nenhum botão isolado chamado apenas `PDF` permanece no grupo neutro.
