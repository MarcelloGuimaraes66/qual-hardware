# Spec — Perceptrum capacity calibration v2

## Requirements

- Export `qual-hardware-calibration-plan/1.0.0` for quick (120/300/180 s) or full (600/2400/600 s) local-only runs.
- Import `qual-hardware-local-calibration/1.0.0` with exact fingerprint, stage metrics, phase metrics, loopback proof and zero external requests.
- Import signed `qual-hardware-evidence-catalog/2.0.0` observations; retain existing signed hardware snapshot support.
- Generate `qual-hardware-capacity-prediction/1.0.0` records with anchors, ratios, margin, range, bottleneck, confidence and provenance.
- Export reports as `capacity-recommendation-export/2.3.0`.
- Interpret inference FPS as requested/effective values; effective video inference is 1–5, while RTSP FPS remains independent.
- Return six distinct candidate designs when six compatible qualified systems exist, ordered by ascending project cost with Intel/NVIDIA preference used only as a tie/diversity policy after capacity safety.

## Errors and safety

- Reject calibration with external network attempts, remote provider, media, credentials, incompatible workload/build, invalid units or non-loopback RTSP/AiQ origins.
- Reject evidence with missing source/version/configuration, duplicate identity or incompatible benchmark direction/version.
- Confidence A requires two comparable anchors and safe leave-one-out evidence; B allows one/adjacent anchor; C/reference-only covers cross-architecture or incomplete data.
- Existing records and tables must never be deleted. Snapshot application uses upsert and marks snapshot membership separately.

## Platform equivalence

- Shared JSON contracts, runner protocol and formulas are OS-neutral.
- macOS is validated live. Windows/Ubuntu paths are built/tested in native CI and require later physical homologation.
