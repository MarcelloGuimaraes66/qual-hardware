# Code design — Perceptrum capacity calibration v2

- `src/shared`: additive versioned calibration/evidence/prediction types and Zod schemas; legacy scenario defaults preserved.
- `src/engine/calibration.ts`: stage mapping, proportional prediction, conservative aggregation, confidence/margin assignment and leave-one-out diagnostics.
- `src/server`: append-only store methods, calibration import/plan/evidence/prediction endpoints and report enrichment.
- `src/web`: calibration center, explicit RTSP versus AiQ FPS labels, anchor/evidence status and extrapolation details.
- `contracts`: machine-readable workload v2 and calibration schemas/examples used by both desktop products.
- Tests: deterministic formulas, bad provenance/version, offline gate, append-only SQLite, API import/export, six-design ordering and report v2.3.

Internal flow: scenario → workload demand → compatible catalog → local/published evidence lookup → per-stage anchor ratio → empirical/safety derating → minimum bottleneck → six candidate presentation. Absence of evidence never upgrades an estimate.

Security: only loopback origins are accepted in calibration; media and credentials are forbidden; signed snapshots retain Ed25519 verification; payload limits are route-specific.
