# Specification — Runtime intake provenance v2

## In scope

- `qual-hardware-calibration-asset-intake/2.0.0` and template `2.0.0`.
- Per-asset `sourcePackages[]` with absolute path, locked SHA-256 and locked size.
- Per-companion `sourcePackageSha256`.
- Exact set/duplicate/coverage/integrity checks before installation.
- Explicit failure for intake/template v1 with migration guidance.

## Preserved behavior

- Nine exact runtime IDs, three targets, CycloneDX/license evidence, destination confinement, symlink rejection, atomic staging/backups, 50 GiB reserve and no network acquisition.
- Runtime assets remain candidates until license, SBOM, package and physical gates approve the resulting manifest hash.

## Failure behavior

- Missing, duplicate, extra, incorrectly sized or hash-mismatched source package: reject before mutation.
- Companion mapped to an unknown package or missing a required source group: reject before asset reads.
- Direct model different from the locked file, or pre-pinned artifact different from its manifest hash/size: reject.
- v1 input: reject with `calibration_asset_intake_v1_source_provenance_required`.

## Platform equivalence

The same schema and verifier apply to `win32-x64`, `darwin-arm64` and `linux-x64`; only locked package metadata and companion extensions differ.
