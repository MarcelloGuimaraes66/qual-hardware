# Planning — Runtime intake provenance

## Task and current state

Continue T4.1 by making offline runtime provisioning prove that operator-supplied files correspond to the immutable packages in `asset-sources.lock.json`. Commit `f0a1dd4` and Desktop CI `29940061181` are green on Windows, Ubuntu and macOS, but intake v1 only hashes the delivered runtime files and does not verify their source packages.

## Goal and acceptance

- Version the intake contract without silently accepting v1 as equivalent.
- Require exact SHA-256 and size for every unique remote source package.
- Bind every auxiliary library group to its declared source package.
- Verify direct-file models against the lock and pre-pinned first-party artifacts against the manifest.
- Remain offline, fail-closed and cross-platform; preserve atomic installation and disk reserve behavior.
- Pass adversarial tests, typecheck, build, source audit and native CI 3×3.

## Risk, scope and budget

T4. Allowed: provisioning contract, CLI template, tests and operator documentation. Untouched: kernel, UI, SQLite, recommendations, Perceptrum and active runtime. Change budget: targeted redesign. Rollback: `f0a1dd4` plus existing feature flag.
