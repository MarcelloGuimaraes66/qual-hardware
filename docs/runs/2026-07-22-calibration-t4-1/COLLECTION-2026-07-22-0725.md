## Proven behavior

The automated collection scenario creates ten separately identifiable machine results, signs them, packages them into one `.qhcalset`, imports them into an installation with no source sessions, confirms the producer identity, and stores ten immutable results. Collector status distinguishes physical machine identities from distinct hardware configurations.

The collector then exports its consolidated logical database as a new `.qhcalset`. The outer collection has the collector's signature, while every embedded `.qhcal` retains its original producer signature and digest. No SQLite file is copied or merged.

Duplicate IDs with equal digests are recorded and ignored. Equal IDs with different digests are recorded as conflicts. Revoked, unmapped, invalid, and untrusted evidence remains available for diagnosis but is excluded from commercial recommendation.
