# Specification — Calibration T4.1

## In scope

- `CalibrationSessionProgress v2`, lifecycle-aware temporary manifest v2, checkpoints/lineage, idempotent cancel/resume APIs.
- Ed25519 device identities and bounded `.qhcal`/`.qhcalset` packages.
- Additive calibration extension schema v2 and collector status.
- Exact-evidence recommendation precedence and immediate deferred catalog retry.
- Non-technical UI for ETA, disk, stop, resume, trust, import/export, and collection status.
- Candidate-only runtime source lock, offline operator intake, atomic provisioning, and mandatory reserve admission.

## Out of scope

- Running or modifying Perceptrum.
- Uploading calibration evidence.
- Claiming physical validation on unavailable Windows/Ubuntu hardware or without approved commercial runtime assets.
- Merging SQLite files or deleting historical records.

## Preserved behavior

- Existing SQLite v9 data and legacy read paths.
- Loopback-only desktop API, renderer CSP/sandbox, existing reports and catalog signing.
- Diagnostic quick tests fail closed for purchase decisions.

## Failure behavior

- Disk reserve breach cancels the active phase safely, persists checkpoint/diagnostic, and cleans registered temporary files.
- Cleanup failure preserves the result, shows pending bytes, and permits an isolated retry.
- Unknown device imports require explicit trust; revoked/untrusted results remain diagnostic.
- Duplicate digest is ignored; same ID/different digest is a recorded conflict.
- Incompatible resume is blocked with exact reasons and does not modify the old session.
- Asset acquisition/provisioning is blocked when the source lock is incomplete/unapproved or projected peak would cross the 10–50 GiB reserve. No audit performs a network download.
- Progress received after cancellation/finalization is ignored; worker events are handled in original order.

## Cross-platform contract

The same package schema, signatures, database extension, and API behavior apply to Windows x64, Linux x64, and macOS arm64. Native installers and physical full runs are mandatory before commercial completion.
