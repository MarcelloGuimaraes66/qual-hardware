# Code — Runtime intake provenance v2

## Modules and flow

- `calibrationRuntimeProvisioning.ts`: v2 schemas, deterministic template, source-package set derivation, structural preflight, file-integrity verification and pinned-artifact checks.
- `prepare-calibration-runtime.ts`: unchanged CLI shape; generated template now emits v2 and dry-run summary reports verified package count.
- Provisioning tests: fixture lock/manifest re-hashing plus substitution, omission, duplicate, mapping and legacy-contract cases.

Flow: parse wrapper/intake → verify manifest/source-lock hash → validate exact inventory and package declarations → verify package files → verify assets/evidence/companions → build proposed manifest → atomically apply only when requested.

## Risk and validation

Main risks are contract compatibility and archive reuse across companion groups. v1 fails explicitly instead of being misread; packages are deduplicated by `sha256:size`; each companion group must be represented. Validate locally and in hosted Windows/macOS/Linux producer and reciprocal receiver jobs.
