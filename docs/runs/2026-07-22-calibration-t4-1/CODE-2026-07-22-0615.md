# Code design — Calibration T4.1

## Modules and responsibilities

- `shared/types` and `shared/schemas`: versioned wire/data contracts and strict bounds.
- `calibrationProgress`: monotonic progress, ETA range/confidence, and terminal 100% rule.
- `calibrationTemporaryFiles`: owned-file lifecycle, reserve calculations, phase reclamation, adversarial path checks.
- `calibrationAssetSources` and `calibrationRuntimeProvisioning`: strict immutable candidate inventory, per-target peak/reserve audit, verified operator-supplied offline intake, atomic staging, and manifest backup.
- `tools/telemetry-probe` and `calibrationTelemetry`: first-party Go sensor, fixed-argument OS/vendor collection, versioned fail-closed JSON, monotonic throttle-counter reconciliation, and strict target/schema/quality validation.
- `calibrationKernelProtocol/Service/Worker`: checkpoint acknowledgement, heartbeat, cooperative cancellation, child termination, and resume seed.
- `calibrationExchange`: canonical gzip, Ed25519 identity/signature, import previews, collections, compression limits.
- `store` and schema: append-only checkpoints, lineage, identities, provenance, import/export audit, snapshots.
- `app`: loopback APIs, transaction ordering, trust workflow, resume compatibility, post-import recalculation.
- `engine/calibration` and `capacity`: exact evidence precedence and distinct-system extrapolation.
- `web/App`: elapsed/remaining display, cancellation, resume, binary import/export, trust and consolidation controls.

## Internal flow

Worker emits progress and checkpoint candidates. The service persists/acknowledges checkpoints before phase files become reclaimable. The store commits results before final evidence and cleanup. Exchange validates byte limits, schema, digest, signature, identity state, duplicates, and conflicts before append-only import.

Worker messages are persisted through one ordered queue. Progress callbacks refuse cancelling or terminal sessions, preventing late progress from racing a cancellation and resurrecting the session. The runtime manifest cryptographically references the asset-source lock, and both files are shipped outside ASAR on all targets.

Cancellation is authoritative over subprocess exit errors and late results. A process terminated because the user pressed Stop cannot be reclassified as a hardware failure, and a result that arrives after cancellation is not committed as purchase evidence.

## Validation plan

Targeted unit/adversarial tests, full Vitest suite, typecheck, build, dependency audit, packaged macOS smoke, structural Windows/Linux packaging, diff check, and source scan for forbidden Perceptrum/egress paths. Native Windows/Linux and physical three-platform calibration remain explicit external gates.
