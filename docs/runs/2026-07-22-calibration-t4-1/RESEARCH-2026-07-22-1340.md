# Pesquisa — arquivo ausente após calibração no macOS

## Causa confirmada

Existem dois destinos divergentes no aplicativo:

1. `CalibrationKernelService.persistEvidence()` grava o arquivo permanente em `calibrationEvidenceDirectory`.
2. `GET/POST /api/calibration-sessions/directory` resolve `Documentos/Qual Hardware/Calibracoes` sem considerar `calibrationEvidenceDirectory`.

No Electron, `calibrationEvidenceDirectory` aponta para `userData/calibration-evidence`. Portanto, o Finder abre uma pasta diferente daquela onde o kernel gravou a evidência.

Além disso, `CalibrationExchangeService.exportRun()` já cria e valida um `.qhcal` assinado, mas só era chamado pela rota acionada manualmente pelo botão de exportação. A conclusão do worker persistia o resultado no SQLite e a evidência compacta, porém não criava automaticamente o pacote portátil esperado para reunir testes de outras máquinas.

## Consequência observável

- A pasta aberta pode estar vazia mesmo depois de um teste bem-sucedido.
- O caminho mostrado na interface pode apontar para um arquivo inexistente.
- O usuário precisa acionar uma exportação adicional para obter o `.qhcal` portátil.

## Estratégia escolhida

- Usar um único diretório permanente como autoridade para escrita, exibição e abertura.
- Criar e verificar o `.qhcal` assinado antes do commit final da sessão.
- Tornar a criação idempotente: um arquivo existente só é reutilizado se a assinatura, o identificador e o digest forem válidos.
- Não sobrescrever conflitos e não remover evidências antigas.
- Manter a exportação manual somente para baixar uma cópia adicional.
