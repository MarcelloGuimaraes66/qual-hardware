# Research — Runtime intake provenance

## Options

1. Add optional provenance fields to intake v1. Rejected: old files could still appear current and omission would remain ambiguous.
2. Introduce intake v2 with exact source-package evidence and an explicit v1 migration error. Selected: auditable and fail-closed without network access.
3. Download and extract upstream packages automatically. Rejected: violates offline/operator-supplied policy and expands archive-extraction and licensing risk.

## Decision

Intake v2 lists every unique locked source package by absolute local path, expected SHA-256 and size. Companion files identify the package hash they came from. Direct files are compared to the lock; artifacts already pinned in the manifest are immutable. Archives are verified but not retained or copied into the runtime. This proves the supplied package set while keeping installation size bounded. Human license/SBOM/physical approval remains mandatory.
