# Validação — resultado automático da calibração

## Prova da correção

- Teste de ponta a ponta cria uma sessão interna, conclui o resultado, confirma `<runId>.qhcal` no diretório permanente, valida a assinatura, verifica o evento no banco e confirma que a API abre exatamente a mesma pasta.
- O teste remove somente seu próprio arquivo temporário para simular uma instalação antiga e comprova a recuperação automática a partir do SQLite.
- O download manual reutiliza o arquivo recuperado sem criar duplicata.
- Um resultado importado sem sessão local continua sendo reexportável.
- A regressão do macOS com `formFactor: "unknown"` está coberta no mesmo ciclo automático.

## Prova de não regressão

- `npm test`: 25 arquivos e 207 testes aprovados.
- `npm run build`: typecheck do renderer, servidor e ferramentas; Vite e TypeScript aprovados.
- `npm run audit:source`: `externalSourceAccess: false`.
- `git diff --check`: aprovado.

## Limitação de ambiente não relacionada à correção

O manifesto de ativos offline continua marcando `readyForProvisioning: false` porque os binários/modelos físicos completos não estão presentes nesta máquina. Isso não afeta a criação, assinatura, persistência, recuperação ou localização do arquivo de resultado corrigidas aqui.
