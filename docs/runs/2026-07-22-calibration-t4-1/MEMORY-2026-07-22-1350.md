# Memória — resultado portátil automático

- Causa raiz: o kernel gravava em `userData/calibration-evidence`, enquanto a interface abria `Documentos/Qual Hardware/Calibracoes`.
- Segunda causa: o `.qhcal` portátil só era criado pelo botão manual de exportação.
- Decisão: `calibrationEvidenceDirectory` é a autoridade única para gravar, mostrar e abrir resultados.
- Invariante: uma sessão autônoma não chega a 100% sem um `.qhcal` assinado, persistido e relido com sucesso.
- Recuperação: resultados autônomos locais existentes no SQLite podem recriar o pacote ausente; conflitos nunca são sobrescritos.
- O botão manual gera apenas outra cópia para conveniência.
- Evidências compactas, pacotes importados e arquivos conflitantes continuam preservados.
