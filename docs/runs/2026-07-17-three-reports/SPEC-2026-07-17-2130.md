# Specification

## Consolidated exports

- Keep the existing export URL compatible, but resolve the complete current recommendation set from the selected recommendation's scenario and revision.
- JSON schema becomes `capacity-recommendation-export/2.0.0` with an ordered `recommendations` array of length three.
- PDF includes a comparison page followed by a clearly separated detailed section for each policy.
- XLSX includes a three-policy summary plus BOM, nodes, calculations, quotes, assumptions/evidence and workload distribution tagged by policy.
- Download filename becomes `qual-hardware-3-configuracoes.<format>`.

## Catalog manager

- **Atualizar hardware** is always visible and opens a modal.
- Configuration accepts an optional HTTPS/loopback catalog URL and a required Ed25519 public key; it is persisted locally outside the repository.
- Manual import accepts a signed JSON snapshot and uses the same configured public key.
- Online refresh and signed import atomically replace hardware and prices only after validation.
- The UI exposes current version/source, configured URL and whether a verification key exists, but never returns private signing material.
