# Specification

## Three differentiated designs

- Calculate all valid candidates for every policy first.
- Select minimum by lowest credible project cost.
- Select recommended from a different hardware template when a compatible alternative exists.
- Select N+1 from a third template when one exists; otherwise it must still have a distinct solution signature through active/reserve node topology.
- Never sacrifice capacity constraints merely to force visual diversity.

## Cost model

- Add a componentized built-in reference-cost profile for each bundled hardware template.
- Convert USD reference values to scenario currency using an explicitly dated BCB/ECB reference rate.
- Prefer current applicable market quotes over the reference profile.
- Expose price basis, observation date, component lines, per-node total, node count and project total.
- Reference ranges use -10%/+15% around the componentized median and remain marked “purchase quotation required”.
- Taxes, shipping, licenses, energy and TCO remain excluded.

## Reports

- PDF: comparison page must show three distinct solutions and numeric project totals; each detail section must show an itemized per-node cost and total project cost.
- XLSX: summary and BOM must contain numeric cost columns, currency and basis.
- JSON: bump export schema and include the full price breakdown.
