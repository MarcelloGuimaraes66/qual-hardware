# Código - restauração do renderizador histórico

## Forma da implementação

- Criar módulo dedicado ao PDF histórico, transcrito do gerador do commit `0846ff7`.
- Manter helpers locais de ordenação, narrativa, preços, plataforma, memória de GPU e layout para impedir que futuras mudanças no JSON/XLSX alterem o PDF histórico.
- Fazer `pdfReport` delegar somente a esse módulo.
- Manter o restante de `reports.ts` funcional e sem mudança de contrato.

## Riscos

- Campos novos podem aumentar o número de páginas sem mudar a estrutura.
- Um helper compartilhado posterior pode alterar silenciosamente o texto; por isso o renderizador será autocontido.
- Dados atuais diferem dos dados de 18/07/2026; a comparação deve avaliar estrutura e layout, não valores antigos.

## Validação

- Teste de estrutura e ausência das seções rejeitadas.
- Geração de amostra e extração textual.
- Renderização integral em PNG e inspeção visual.
- Testes, typecheck, build, pacote e smoke desktop.
